# Demo Paths

These paths group existing fixtures, commands, and report snapshots into three
copyable workflows. Use them when you want to understand the suite by release
task instead of by package name.

| Demo path | Shows | Start here |
| --- | --- | --- |
| Tiny mobile release | Export presets, input coverage, mobile UI metadata, localization stress inputs, visual-smoke planning, and a release dashboard. | [`tiny-mobile-release`](tiny-mobile-release/README.md) |
| Tiny content game | Content graph checks, pack/mod manifests, save migration fixtures, and scenario evidence. | [`tiny-content-game`](tiny-content-game/README.md) |
| Tiny runtime regression | Scenario comparison, retry/flake evidence, runtime telemetry budgets, timeline reports, and dashboard trend inputs. | [`tiny-runtime-regression`](tiny-runtime-regression/README.md) |

The report snapshots referenced here are collected in the
[sample report gallery](../../docs/assets/sample-reports/README.md).

For a separate tiny Godot project that runs the toolkit actions in GitHub
Actions, see the
[public CI demo](https://github.com/NonniGB/godot-production-toolkit-demo-ci).
