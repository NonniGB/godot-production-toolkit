# Support

## Before opening an issue

Run the relevant command with `--format json --fail-on none` when supported. Attach the command, summarized output, and a minimal fixture.

## Where to ask

Use GitHub issues for bugs and focused feature requests. Use discussions or community posts for broader workflow questions once the repository has discussions enabled.

## What to include

- Tool name and version.
- Python version.
- Godot version when relevant.
- Operating system.
- Exact command.
- Expected result.
- Actual result.
- Minimal project or fixture when possible.
